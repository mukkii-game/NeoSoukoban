# CLAUDE.md — あなふさぎのよる 開発ルール

## プロジェクト概要
絵文字ベースの短編Web パズルゲーム『あなふさぎのよる』。
見た目は倉庫番だが、実は「ルールの読み違いに気づく」知識アンロック型パズル。
仕様は `GAME_SPEC.md`、進捗と判断理由は `DEVELOPMENT_LOG.md` を参照。

## ファイル構成
- `index.html` — ゲーム本体のシェル（HTML + CSS）。GitHub Pages のエントリポイント
- `levels.js` — レベルデータ（純データ。ロジックを書かない）
- `solutions.js` — 模範解答（自動生成。`node tools/solve.mjs --emit`。手で編集しない）
- `engine.js` — 純粋ロジック（状態遷移のみ。DOM/audio 禁止。Node からも読める UMD 形式）
- `main.js` — UI・描画・入力・音・セーブ（ブラウザ専用コードはすべてここ）
- `tools/solve.mjs` — BFS ソルバー。全レベルのクリア可能性を機械検証する
- `NeoSoukoban.html` — 旧ゲーム（レガシー。触らない・消さない）

## 開発ルール
1. **決定論を守る**: engine.js に `Math.random()` / `performance.now()` / リアルタイム要素を入れない。
   すべての状態変化はプレイヤーの1入力（移動 or 待機）から決定的に導出される。
2. **engine と UI の分離**: ゲームルールの変更は必ず engine.js のみで行い、main.js は events を描画するだけ。
   これを崩すとソルバー検証が嘘になる。
3. **レベルやルールを変更したら必ず `node tools/solve.mjs --emit` を実行**して
   全レベル SOLVABLE の確認と `solutions.js`（ギブアップ用模範解答）の再生成を行う。
   気づきレベル（aha レベル）は「ナイーブルールでは UNSOLVABLE」の検証も行う。
3b. **可解 ≠ 発見可能。** コアの気づきごとに「ナイーブなプレイヤーが最初に試す操作」を
   列挙し、その結果を人間視点でトレースする。間違った試行が安全に情報を返すこと
   （罰が即敗北にならないこと）を確認する。
4. **Undo は完全スナップショット**: 差分ではなく全状態を積む。湧きカウンタ・敗北状態も戻る。
5. 外部依存ゼロ。CDN 禁止。`file://` で index.html を開いて動くこと。
6. スマホ対応必須: スワイプ + 画面ボタン。`touch-action` に注意。
7. コンソールエラーゼロを維持。

## 設計方針（ゲームデザイン）
- ルールは少数・深く。新ギミック追加より「既存ルールの見方が変わる」配置を優先する。
- テキストで教えない。レベル名・配置・詰まり体験で気づかせる。
- 敵はパズルピース。反射神経を要求しない（完全ターン制）。
- 理不尽な詰みを作らない: Undo と Retry はいつでも1操作で届く位置に置く。

## 検証コマンド
```
node tools/solve.mjs          # 全レベルの機械検証
node tools/solve.mjs --emit   # 検証 + solutions.js 再生成（レベル/ルール変更時は必須）
node tools/browsertest.mjs    # E2E（puppeteer-core を npm install した dir を cwd に）
```
