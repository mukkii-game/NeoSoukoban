# GAME_SPEC.md — 『ワルぶるネコのよる』仕様書 (v1.9)

## コンセプト
プレイヤーは「はーん、倉庫番ね」と思って遊び始める。
しかし本当のルールは少しずつ違う。ゴールに見えるものは **穴** であり、
箱を「置く」のではなく穴を「塞ぐ」ゲームだった——さらに、壁も、おばけも、
穴そのものさえも「押せる」——と、遊びながら気づいていく **知識アンロック型パズル**。
20〜30分の短編。ひと晩（PM9:00〜夜明け）の物語、全21面。

- タイトル: **ワルぶるネコのよる**
- 主人公: ワルぶってるネコ 😼。まよなかの倉庫番バイトで、おばけだらけの夜に
  「べつに、こわくないし」と奮闘する
- 見た目: 絵文字 + CSS のタイニーなドット風。夜の倉庫。

## コアルール（真のルール）
1. グリッド上の完全ターン制。プレイヤーが1歩動く（or 待つ）と世界が1ターン進む。
2. **かべは、実は押せる**（1マス押し。見た目は普通の壁と完全に同じ）。
   - 非 wrap 面の外周のかべは「外側に立てない」ため幾何的に絶対動かない。
     これが実質的な盤の枠になる（枠は盤面の影で視認できる）。
   - `X` の **おせないかべ** だけは押せない。見た目はほんの少しだけ違う
     （わずかに暗く、かすかな鋲がある）。
3. **箱 📦**: 1マス押し。チェーン押し不可。
4. **おばけ 👻**:
   - 毎ターン、プレイヤーに1歩近づく（決定論的）。
     **(a) 連結追従（ぱっくまんの列）**: プレイヤーが空けたマスのすぐ隣にいる子は
     そのマスへ「ぴったり」入り、その子が空けたマスにさらに隣の子が入り……と、
     **遠ざからない限り列ごと順々に詰めてくる**。
     → 歩くだけで自然にきれいな列ができる。plop の布石も「道を敷く」感覚。
     **(a2) きんぎょのフン（隊列保持）**: プレイヤーから途切れずつながっている子は、
     前が動かない限りその場で待つ（プレイヤーが止まっても列が勝手に崩れない）。
     つながりが切れた子だけが (b) の鏡像軸で動く。
     **(a3) 列の完全追従**: 列（プレイヤーと連結）のメンバーは道なりに無条件で
     ついてくる（曲がり角で直線距離が一時的に離れても切れない）。
     列に属さない外部の子だけ「入ると遠ざかるなら合流しない」ガードが効く。
     **(b) 鏡像軸**: 離れている子は、プレイヤーが最後に動いた軸を優先して合わせてくる
     （横に動けば横座標を、縦に動けば縦座標を）。
     その軸の差が0のとき/未移動時は距離の大きい軸を優先。
     塞がれていたらもう一方の軸。両方ダメなら停止。
   - **ループ面での追跡視野**: プレイヤーとの距離（マンハッタン）が3以内のときだけ
     ワープ越しの最短方向で追う。遠いときはワープを忘れてまっすぐ向かう
     （「近づこうとしたら逆に離れていく」直感違反の防止）。
   - **プレイヤーのマスには入れない**（隣で止まる）。**このゲームに敗北は存在しない。**
   - **単体のおばけをプレイヤーが直接押すと、先が穴でも壁でも箱でも必ずよける**:
     半マス押し出されて斜めに傾いてから、するりと元の位置へ戻る
     （「ヒョイ」演出）。よけるのに精一杯で、そのターンは動けない。押し手の
     後ろ（プレイヤーがいた場所）が空いたままなので戻れる＝逃げ場がある、
     という理屈。2体以上の列が壁・箱に阻まれたときは（単体と違い）本当に
     ブロックされて「ドン」と押し返される（bonk）。
   - **戻れず捕まる（穴なら落ちて栓になる）のは、戻り先が塞がっている場合だけ**:
     (a) **2体以上が縦横に列をなしているとき**（列ごとまとめて押せる。
         列の先が穴なら先頭の1体が落ちる。他のメンバーが戻り先を埋めるので
         列全体が戻れない。それ以外のメンバーは1マス詰めるだけ）
     (b) **箱でおばけ（の列）を押したとき**（箱自身が戻り先を埋めてしまうので、
         単体のおばけでも戻れない。箱は逆にはおばけに押されない
         — おばけが箱にぶつかると半分重なる程度でヒョイと引っ込む）。
   - つまり「並ばせる」「箱で押す」「穴の前に追い込む」という位置取りがパズルになる。
   - **勢い（momentum）**: 列を押していって最後の1体が単体になっても、
     **同じ方向へ押し続けている（連打・長押しし続けている）限り、床の上では
     よけない**（すぐ戻らず前進を続けられる）。方向を変えたり、待つ・他の
     操作を挟むと勢いが切れ、次に単体を押すと素直によける。
     **ただし勢いだけでは穴に捕まらない**: 単体のおばけが穴に触れたときは、
     勢いの有無に関わらず必ずよける（穴に捕まるのは2体以上の連結列・箱に
     押された場合・既にばたばた中の場合だけ、という原則を維持するため）。
     箱を使えば、途中で止まっても最後の単体を塞げる。
     **列のだれかが穴に触れた（プラグ／ばたばた化した）ターンは、その巻き添えで
     単体になった残りの子に勢いはつかない**。次に押すと素直によける（＝
     連結列から1体キャッチしたら、その瞬間に生き残りへの押しっぱなし特典は
     リセットされる。あくまで穴に一切関わらない純粋な床の連結押しだけが
     勢いを生む）。
   - **ばたばた確定（連続した穴での猶予）**: 2体以上の連結列を押していて、
     先頭が穴に着地した瞬間、その先にもう1つ開いた穴が連続していれば、
     先頭は即プラグせず「ばたばた」状態（未確定・回転気味）になる。**次の
     ターンに動かされなければ、そこで確定して落ちる**。動かされた場合
     （＝さらに押された）は、ばたばた中の子はさらに奥の穴へ、新しく列に
     加わった子は手前の空いた穴でまた「ばたばた」し始める。両方ともばたばた
     中の状態で放置（何か別の操作をする）すれば、その瞬間に両方同時に
     確定して落ちる。3体以上の列・3連続以上の穴でも同様に伸びる。
     穴が孤立している（連続していない）場合や単体の即席キャッチ（箱・勢い
     経由）ではこの猶予は発生せず、従来どおり即座に確定する。
     実時間（ミリ秒タイマー）は一切使わない完全ターン制の機構
     （CLAUDE.md の決定論ルールに準拠。ばたつきアニメの表示時間はUI側の演出
     のみでゲーム状態には影響しない）。**ただしUI側の便宜として**、
     ばたばた発生後0.5秒プレイヤーが何も操作しなければ、main.js が内部的に
     「待つ」を1回自動発行して確定させる（じきゅうじそくの孤立穴キャッチと
     体感を揃えるため）。タイムアウト前に何か操作すればこの自動待ちは
     キャンセルされる。エンジン自体は常に明示的な1入力からしか状態を
     進めないため、決定論・ソルバー検証には一切影響しない。
   - 押されたおばけは目を回し（💫）、そのターンは動けない。
   - おばけは開いた穴・壁・箱・他のおばけには入らない（自分からは穴に落ちない）。
5. **穴 🕳**: このゲームの「ゴール」。押せるもの（かべ・箱・おばけ）が入ると
   落ちて塞がる（消滅し、穴は床になる）。**すべての穴を塞ぐとクリア**。
   **開いた穴そのものも押して動かせる**（押し先が壁・オブジェクト・別の穴のときは
   動かず、プレイヤーは怖くて入れない 💦）。塞がった穴の上は歩ける。
6. **湧き**: 開いた穴は N ターンごとに 👻 を1体産む（spawnInterval。レベルごとに個別設定、
   未指定=0 なら湧かない）。湧く1ターン前に 👀 が出る。塞がれば湧かない。
   同時存在数は ghostCap で制限（レベルごとに個別設定、未指定時デフォルト4）。
   `spawnOnce: true` を指定した穴は**生涯で1体しか湧かない**（湧いた個体が
   逃げても再湧きしない。じきゅうじそくで使用）。これらはすべて **レベル単位の設定**
   であり、ゲーム全体で共通のグローバル定数ではない。湧いたおばけはその穴の
   マスに立つが、穴自体はまだ開いたまま（塞がっていない）。まだ動いていない
   その瞬間に箱・かべを押し込んでおばけを押しのければ、通常どおり箱・かべが
   その穴に落ちて塞ぐ。
7. **ループ面**: `wrap: true` のレベルは盤面の上下左右がつながっている。
   プレイヤー・押されたもの・おばけの追跡すべてに適用（最短方向で追う）。
   ループ面では外周固定則が働かないため、静的壁はすべて `X` で書く。

## ターン順序（決定論）
1. プレイヤー入力（移動 / 押し / 穴押し / 待機）。
   **できない操作（壁ドン・押せない・よけられる・こわい）でも世界は1ターン進む**
   ＝壁にぶつかるのが実質の「待つ」。Space の明示的な待機も残してある（補助）。
2. 落下判定 → 全穴が塞がればその瞬間クリア。
3. おばけ移動（盤面の読み順 y→x で1体ずつ。目を回している個体はスキップ。
   プレイヤーのマスには入らない）。
4. 湧きカウンタ減算 → 0 の穴から 👻 生成（そのターンは動かない）。

## レベルデザイン指針（囲まれ対策）
- おばけが複数いる広い平面には、**引っ掛け用のブロックを最低1つ**置く
  （柱に敵を引っ掛ければ、手数はかかるが逃げ道がある状態を保つ）。
  完全に詰んでも Undo/Retry/💡 で復帰できるが、「逃げられなくもない」余地を残す。

## 必須機能
- キーボード: 矢印/WASD 移動（ぶつかっても1ターン進む）、Space 待機（補助）、
  Z Undo、R リトライ、Esc レベル選択
- スマホ: スワイプ移動 + 画面下 D-pad（中央 = 待機）+ Undo/Retry ボタン
- タイトル / レベル選択（順次アンロック）/ クリア演出 / 短いエンディング
- 💡 模範解答の自動再生（ギブアップ）: **まず「いまの状態から」予算つき探索で解き**、
  だめならグリーディ段階探索。ここから詰みと証明されたら赤警告。
  探索不能な巨大局面のみ最初から再生。再生速度 280ms/手。
  最初から用の解データは `node tools/solve.mjs --emit` が `solutions.js` に自動生成
- ★ランク: クリア手数 ≦ 最短+2 → ★★★ / ≦ 最短+10 → ★★☆ / それ以外 → ★☆☆。
  自己ベストは localStorage（best）。「最短」= solutions.js の par
  （witness 面は最良既知手数）。レベル選択とクリア画面に表示、EDで合計を発表
- **Perfect! バッジ**: 自己ベスト手数が par 以下（＝ソルバーの最短、witness 面では
  それを上回る発見）のときだけ、きらめき付きの金グラデーション文字で表示。
  クリア画面では★の並びのすぐ上（隣接）に表示、専用ファンファーレ音つき。
  レベル選択でも該当カードの★を金色に発光させ、星の下に明示的な文字ラベル
  「Perfect!」を添えて一覧から分かるようにする（小さなきらめきバッジ✦のみだと
  分かりづらいとのフィードバックで文字表示に変更。isPerfectCleared: SAVE.best[id]
  <= par で判定）
- カメラ: セルサイズは全面で統一（縮小しない）。盤が画面より大きい面は
  プレイヤー中心に盤がパンする（クランプつき）
- 🔓 検証用の全レベル開放トグル（レベル選択画面の下部）
- **行き詰まり検出（毎ターン・分割実行）**: 入力の 250ms 後から、現在の状態の BFS を
  約11ms のスライスに分けてバックグラウンドで実行（予算 220,000 状態。フレームを
  止めない。新しい入力が来たら中断してやり直し）。全探索し尽くして解が無ければ
  「詰み」を証明し **⚠️ 赤字の警告**で Z/R を提案する。予算切れは沈黙＝誤報を出さない。
  解ける状態に戻れば警告は消える
- 視認性: おばけの移動は複数いるとき 50ms ずつの時間差 + 移動元に残像 👻 を残す。
  押せなかった壁/箱はプルッと震える。**一度でも壁を押せたら（SAVE.wallSeen）、
  以後すべての押せる壁にうっすら取っ手あとが見える**（発見前の嘘は守る）
- クリア画面のキー操作: Enter/移動キー = つぎへ、R = もういちど、Space/Esc = レベルせんたく
- ぎょうれつ押しの演出: 手前から順に「ゴン！ゴン！」と時間差（85ms/体）で押される。
  単体のよけは「スカッ」（1コマ出て戻る）、穴落ちは「にゅーっ」と沈む
- 音: WebAudio のミニ SE + 夜のループ BGM（A→A→B 構成。プレイ中のみ）+ ミュート永続化
- 常時アニメ: ネコ/おばけ/箱のゆらぎは個体ごとに位相と速度をずらす（描画のみ乱数可）。
  押せる壁は動かない見た目を保つ（揺れたら正体がバレる）
- セーブ: クリア状況を localStorage に保存
- コンソールエラーなし。file:// 直開きで動作。GitHub Pages 対応（index.html）

  実装する設計（盤内に「偽UI行」を持つ専用レベル）を将来課題として保留
